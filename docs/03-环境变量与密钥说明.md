# 环境变量与密钥说明

这份文档说明每个变量放在哪里、怎么获取、是不是必须。

## 1. 文件和平台的区别

本项目有三类配置位置：

- `.env.local`：本地开发用，复制 `.env.example` 得到，不要上传 GitHub。
- `.dev.vars`：Wrangler/OpenNext 本地预览用，复制 `.dev.vars.example` 得到，不要上传 GitHub。
- `wrangler.toml`：Cloudflare Worker 配置，可以上传 GitHub，但不要写真实 Secret。
- Cloudflare `Variables and Secrets`：生产环境变量和密钥，Secret 不会出现在 GitHub。

`.gitignore` 已经忽略：

```text
.env
.env.*
.dev.vars
```

但保留：

```text
.env.example
.dev.vars.example
```

| `R2_SECRET_ACCESS_KEY` | 不公开 | Cloudflare Secret | R2 Secret Access Key |

## 3. R2 公开访问变量

| 名称 | 示例 | 是否必须 | 说明 |
| --- | --- | --- | --- |
| `R2_PUBLIC_BASE_URL` | `https://pub-xxx.r2.dev` | 推荐 | R2 公开访问地址 |
| `R2_CDN_BASE_URL` | `https://cdn.example.com` | 推荐 | 自定义 CDN 或公开域名 |
| `NEXT_PUBLIC_R2_CDN_BASE_URL` | `https://cdn.example.com` | 推荐 | 前端读取的 R2 CDN 地址 |

如果你还没有公开域名，可以先留空。文件上传和签名下载主要依赖 R2 API 密钥；但音乐、图片直链、公开预览体验会受影响。

## 4. Cookie 和域名

| 名称 | 示例 | 建议 |
| --- | --- | --- |
| `SYSTEM_COOKIE_DOMAIN` | `.example.com` | 新手留空 |

留空时，登录 Cookie 只对当前域名有效，最不容易出错。

只有当你明确要让多个子域名共享登录态时才设置，例如：

```text
.example.com
```

如果设置错了，会出现登录成功后马上退出、刷新后未登录等问题。

## 5. 定时清理 Secret

| 名称 | 用途 |
| --- | --- |
| `TRADE_LOG_RETENTION_SECRET` | 交易日志清理接口密钥 |
| `SYSTEM_RETENTION_SECRET` | 系统综合清理接口密钥 |
| `CONSULT_RETENTION_SECRET` | 咨询消息清理接口密钥 |

生成方式：

```powershell
[guid]::NewGuid().ToString("N")
```

设置方式：

```powershell
npx wrangler secret put TRADE_LOG_RETENTION_SECRET
npx wrangler secret put SYSTEM_RETENTION_SECRET
npx wrangler secret put CONSULT_RETENTION_SECRET
```

## 7. 可选 WebRTC TURN 变量

只有咨询音频通话在严格网络环境下需要 TURN。没有 TURN 时，文字咨询不受影响。

| 名称 | 示例 |
| --- | --- |
| `NEXT_PUBLIC_WEBRTC_TURN_URLS` | `turn:turn.example.com:3478` |
| `NEXT_PUBLIC_WEBRTC_TURN_USERNAME` | `username` |
| `NEXT_PUBLIC_WEBRTC_TURN_CREDENTIAL` | `password` |

多个 TURN URL 可以用逗号分隔。

## 8. 下载相关变量

| 名称 | 默认值 | 说明 |
| --- | --- | --- |
| `R2_DESKTOP_PREFIX` | `desktop` | 桌面客户端下载文件在 R2 中的前缀 |
| `R2_MOBILE_PREFIX` | `mobile` | 移动客户端下载文件在 R2 中的前缀 |
| `IOS_DOWNLOAD_URL` | 空 | 可选 iOS 外部下载地址 |

如果你不发布客户端安装包，可以不配置。

## 9. 登录限流变量

| 名称 | 默认值 | 说明 |
| --- | --- | --- |
| `MIDDLEWARE_LOGIN_RATE_WINDOW_MS` | `60000` | 中间件登录限流窗口 |
| `MIDDLEWARE_LOGIN_RATE_LIMIT_PER_IP` | `80` | 单 IP 窗口内最大请求数 |
| `LOGIN_RATE_LIMIT_STORE` | `d1` | 登录限流存储，生产建议 D1 |
| `LOGIN_RATE_LIMIT_WINDOW_MS` | `600000` | 登录失败统计窗口 |
| `LOGIN_RATE_LIMIT_MAX_ATTEMPTS_IP` | `60` | 单 IP 最大失败次数 |
| `LOGIN_RATE_LIMIT_MAX_ATTEMPTS_EMAIL` | `10` | 单邮箱最大失败次数 |

新手保持默认即可。

## 10. Cloudflare D1 绑定

`wrangler.toml` 必须有：

```toml
[[d1_databases]]
binding = "DB"
database_name = "fxlocus-system"
database_id = "你的 D1 database_id"
```

代码会通过 `DB` 这个 binding 访问 D1。不要改 `binding = "DB"`，除非你也同步修改代码。

## 11. Cloudflare R2 绑定

`wrangler.toml` 必须有：

```toml
[[r2_buckets]]
binding = "R2_ASSETS"
bucket_name = "fxlocus-system-files"
```

代码会通过 `R2_ASSETS` 访问 R2。不要改 `binding = "R2_ASSETS"`，除非你也同步修改代码。

## 12. 怎么判断一个值能不能上传 GitHub

可以上传：

- 数据库名称。
- R2 bucket 名称。
- 网站公开域名。
- `.env.example` 里的空示例。
- `wrangler.toml` 里的非密钥配置。

不要上传：

- `R2_SECRET_ACCESS_KEY`
- `R2_ACCESS_KEY_ID`
- Cron Secret
- `.env.local`
- `.dev.vars`
- 任何真实用户数据导出文件
- Cloudflare API Token

## 13. 修改变量后的生效方式

改了 `wrangler.toml`：

```powershell
git add wrangler.toml
git commit -m "Update Cloudflare config"
git push
```

如果用 Cloudflare GitHub 自动部署，推送后会重新部署。

改了 Cloudflare Secret：

1. 保存 Secret。
2. 重新部署 Worker。
3. 再测试系统。

改了 `.env.local`：

1. 停止本地 `npm run dev`。
2. 重新运行 `npm run dev`。
