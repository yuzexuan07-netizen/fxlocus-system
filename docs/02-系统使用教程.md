# 系统使用教程

本教程从第一次登录开始，说明系统里各角色能做什么，以及后台数据怎么建立。

## 1. 登录地址

中文系统登录地址：

```text
https://你的域名/zh/system/login
```

本地开发地址：

```text
http://localhost:3000/zh/system/login
```

第一次登录使用部署教程里创建的超级管理员账号。

系统右下角默认显示 `fxlocus 开源版 - MIT` 水印，用来标识当前部署的是开源版本。

## 2. 角色说明

系统内置角色：

- `super_admin`：超级管理员，拥有最高权限。
- `leader`：团队长，可以管理自己团队内的用户和内容。
- `assistant`：助教，可以管理自己创建或被分配的学员资料、提交内容。
- `coach`：教练，可以查看和处理被分配学员的学习、交易、文档内容。
- `student`：普通学员，可以学习课程、上传内容、查看通知和资料。
- `trader`：交易员，和学员类似，但用于区分交易相关身份。

第一次部署后只有超级管理员。其它账号需要在后台创建。

## 3. 第一次进入后台

登录超级管理员账号后，系统会自动进入后台首页。

如果没有自动进入，可以访问：

```text
https://你的域名/zh/system/admin
```

后台左侧是功能菜单，右侧是当前功能页面。

## 4. 创建团队长

适合先建立团队结构。

1. 登录超级管理员。
2. 进入后台。
3. 点击左侧 `团队长` 或 `Leaders`。
4. 点击新建按钮。
5. 填写姓名、邮箱、手机号等信息。
6. 设置初始密码。
7. 保存。

创建后，把登录地址、邮箱、初始密码发给团队长。

## 5. 创建助教

助教可以由超级管理员创建，也可以由团队长在自己的权限范围内创建。

1. 进入后台。
2. 点击 `助教` 或 `Assistants`。
3. 点击新建。
4. 填写姓名、邮箱、手机号。
5. 选择所属团队长。
6. 设置初始密码。
7. 保存。

助教账号创建后可以登录系统，处理自己权限内的学员资料和提交内容。

## 6. 创建学员或交易员

1. 进入后台。
2. 点击 `学员` 或 `Students`。
3. 点击新建。
4. 填写姓名、邮箱、手机号。
5. 选择角色或学员状态。
6. 选择所属团队长、助教或教练。
7. 设置初始密码。
8. 保存。

学员登录后通常会进入：

```text
/zh/system/dashboard
```

学员可以看到课程、资料、通知、上传入口等。

## 7. 重置密码

如果用户忘记密码：

1. 管理员进入后台。
2. 打开 `学员`、`团队长` 或 `助教` 列表。
3. 找到对应用户。
4. 进入详情或点击操作按钮。
5. 选择 `重置密码`。
6. 输入新密码。
7. 保存后通知用户新密码。

建议用户第一次登录后到个人资料页修改密码。

## 8. 冻结、删除用户

冻结适合临时禁用账号：

1. 进入用户列表。
2. 找到用户。
3. 点击冻结或状态按钮。
4. 确认操作。

冻结后用户无法正常使用系统。

删除适合清理错误账号。正式运营前请谨慎使用硬删除，因为它可能连同用户相关数据一起清理。

## 9. 课程内容管理

课程用于给学员学习。

1. 管理员进入后台。
2. 点击 `课程` 或 `课程内容`。
3. 新建或编辑课程。
4. 填写课程标题、分组、顺序、说明。
5. 上传课程文件或填写内容。
6. 保存。

课程访问可以通过后台审批或授权控制。学员申请课程后，管理员或团队长在课程申请列表中审批。

## 10. 文件资料管理

系统的资料文件存储在 Cloudflare R2。

使用前确认已经配置：

- R2 bucket：`fxlocus-system-files`
- R2 API 密钥：`R2_ACCESS_KEY_ID`、`R2_SECRET_ACCESS_KEY`
- Worker R2 binding：`R2_ASSETS`

上传资料：

1. 管理员进入后台。
2. 点击 `文件` 或 `Files`。
3. 点击上传。
4. 选择本地文件。
5. 填写文件名称、说明、权限范围。
6. 保存。

授权资料：

1. 进入文件列表。
2. 找到文件。
3. 点击授权或权限。
4. 选择学员或角色。
5. 保存。

学员申请资料：

1. 学员进入 `资料` 页面。
2. 找到需要的文件。
3. 点击申请。
4. 管理员在后台 `文件申请` 页面审批。

## 11. 交易日志和交易策略

学员、交易员、教练等角色可以上传交易相关内容。

学员操作：

1. 登录系统。
2. 进入 `交易日志` 或 `交易策略`。
3. 点击上传或新建。
4. 填写说明。
5. 上传截图、文档或相关文件。
6. 提交。

管理员或教练操作：

1. 进入后台。
2. 打开 `交易日志`、`交易策略` 或相关审核页面。
3. 查看提交内容。
4. 填写审核意见。
5. 通过、驳回或归档。

## 12. 周报

周报用于学员、团队长、助教等周期性提交总结。

提交周报：

1. 登录系统。
2. 点击 `周报`。
3. 填写总结文本。
4. 上传策略、曲线、统计等附件。
5. 点击提交。

审核周报：

1. 管理员进入后台。
2. 点击 `周报`。
3. 按角色或团队筛选。
4. 查看详情。
5. 填写审核意见。
6. 保存审核结果。

## 13. 天梯图和授权

天梯图一般用于展示阶段、等级或榜单图片。

管理员上传：

1. 进入后台。
2. 点击 `天梯`。
3. 上传图片。
4. 设置刷新间隔或展示配置。
5. 保存。

学员申请：

1. 学员进入 `天梯` 页面。
2. 如果没有权限，提交访问申请。
3. 管理员在后台审批。

## 14. 通知

通知用于给用户发送站内消息。

发送通知：

1. 管理员进入后台。
2. 点击 `通知`。
3. 选择接收人。
4. 输入标题和内容。
5. 发送。

全局通知：

1. 进入通知管理。
2. 选择全局通知。
3. 填写内容。
4. 发送给所有目标用户。

用户登录后可以在通知页面查看未读和已读消息。

## 15. 咨询沟通

咨询功能用于系统内用户沟通。

1. 登录系统。
2. 点击 `咨询`。
3. 选择联系人。
4. 输入文字或上传支持的附件。
5. 发送。

可选音频通话依赖浏览器 WebRTC。普通网络可以直接使用；如果用户在严格网络环境下需要中继服务，可以配置 TURN：

- `NEXT_PUBLIC_WEBRTC_TURN_URLS`
- `NEXT_PUBLIC_WEBRTC_TURN_USERNAME`
- `NEXT_PUBLIC_WEBRTC_TURN_CREDENTIAL`

不配置 TURN 时，文字咨询和普通消息不受影响。

## 16. 个人资料

用户可以在个人资料页修改：

- 姓名
- 联系信息
- 密码

如果修改密码失败，请管理员在后台重置密码。

## 17. 后端数据怎么建立

本项目开源包不包含真实数据库数据。你需要自己建立：

1. D1 数据库结构：运行 `npm run d1:init:remote`。
2. 第一个超级管理员：运行 `npm run d1:create-admin:remote`。
3. 团队长、助教、教练、学员：登录后台后创建。
4. 课程、资料、通知、天梯图：登录后台后上传或录入。
5. 学员提交的日志、策略、周报、文档：由实际用户在系统中产生。

数据库表结构在：

```text
d1/schema.sql
d1/migrations
```

真实用户数据不会随开源代码发布。

## 18. 日常维护

建议每周检查：

- Cloudflare Worker 是否有错误日志。
- D1 数据库是否正常。
- R2 存储桶容量是否异常增长。
- GitHub 仓库是否误提交 `.env.local`、`.dev.vars` 或密钥。
- 管理员账号是否使用强密码。

可选定时清理接口：

- `/api/cron/system-retention`
- `/api/cron/consult-retention`
- `/api/cron/trade-logs-retention`

这些接口需要配置对应 Secret，避免被外部随意调用。

## 19. 常见问题

### 登录提示账号或密码错误

检查：

- 是否创建了超级管理员。
- 邮箱是否小写且没有空格。
- 密码是否输错。
- D1 是否绑定到 Worker。
- 是否重新部署了最新配置。

### 文件上传失败

检查：

- R2 bucket 是否存在。
- `wrangler.toml` 的 `bucket_name` 是否正确。
- Cloudflare Secrets 是否设置了 `R2_ACCESS_KEY_ID` 和 `R2_SECRET_ACCESS_KEY`。
- R2 API Token 是否有读写权限。

### Cloudflare 构建失败

检查：

- GitHub 仓库是否包含 `package-lock.json`。
- Build command 是否是 `npm run cf:build`。
- Deploy command 是否是 `npx wrangler deploy`。
- `wrangler.toml` 的 D1 `database_id` 是否已经替换。
- 构建环境是否使用 Node.js LTS。

### Windows 本地 `npm run cf:build` 失败

Windows 本地 OpenNext Cloudflare 构建可能不稳定。新手可以只在本地运行：

```powershell
npm run build
npm run dev
```

生产部署交给 Cloudflare 的 Linux 构建环境执行 `npm run cf:build`。
