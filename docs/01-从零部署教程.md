# 从零部署教程

这份教程按完全新手的顺序写：先安装工具，再在自己电脑运行，再创建 GitHub 仓库，最后部署到 Cloudflare 和自己的域名。

Cloudflare、GitHub 的界面文案可能会轻微变化。如果按钮名称不完全一样，优先找含义相同的入口，例如 `Create`、`New repository`、`Variables and Secrets`、`D1`、`R2`、`Workers`。

## 1. 你需要准备什么

你需要：

- 一台电脑，Windows、macOS、Linux 都可以。
- 一个邮箱，用来注册 GitHub 和 Cloudflare。
- 一个 GitHub 账号，用来保存开源代码。
- 一个 Cloudflare 账号，用来部署网站、创建数据库和文件存储。
- 一个域名。如果暂时没有域名，也可以先用 Cloudflare 分配的 workers.dev 地址测试。

本项目使用：

- Next.js：网站和系统界面。
- Cloudflare Workers：部署后端和网页。
- Cloudflare D1：数据库，只保存你自己部署后的数据。
- Cloudflare R2：上传文件、图片、文档等对象存储。
- GitHub：保存开源代码。

## 2. 安装 Node.js

Node.js 是运行前端项目的基础工具。

1. 打开浏览器。
2. 进入 https://nodejs.org/en/download 。
3. 选择 `LTS` 版本。
4. Windows 用户点击 Windows Installer；macOS 用户点击 macOS Installer。
5. 下载完成后双击安装包。
6. 一路点击 `Next`。
7. 看到许可协议时勾选同意，再点 `Next`。
8. 安装路径保持默认即可。
9. 点击 `Install`。
10. 安装完成后点击 `Finish`。

验证是否安装成功：

1. Windows 按 `Win + R`，输入 `powershell`，按回车。
2. 输入：

```powershell
node -v
npm -v
```

如果能看到版本号，例如 `v22.x.x` 和 `10.x.x`，说明安装成功。

如果提示 `node 不是内部或外部命令`，重启电脑后再试一次。

## 3. 安装 Git

Git 用来把代码上传到 GitHub。

1. 打开 https://git-scm.com/downloads 。
2. 选择你的系统，例如 Windows。
3. 下载 Git 安装包。
4. 双击安装包。
5. 一路点击 `Next`，保持默认选项即可。
6. 点击 `Install`。
7. 安装完成后点击 `Finish`。

验证：

```powershell
git --version
```

如果能看到版本号，例如 `git version 2.x.x`，说明安装成功。

## 4. 在电脑上运行项目

打开 PowerShell，进入项目文件夹：

```powershell
cd e:\fxlocus-system-open-source
```

安装依赖：

```powershell
npm install
```

第一次安装会下载很多文件，可能需要几分钟。安装完成后运行构建检查：

```powershell
npm run build
```

如果最后没有红色错误，并且命令退出，说明代码可以正常构建。

启动本地开发服务：

```powershell
npm run dev
```

看到类似 `http://localhost:3000` 后，打开浏览器访问：

```text
http://localhost:3000/zh/system/login
```

注意：本地页面能打开，不代表已经能登录。登录需要先创建 D1 数据库并创建管理员账号。新手建议先完成 Cloudflare 远程 D1 设置，再登录线上系统测试。

停止本地服务：

- 在 PowerShell 窗口按 `Ctrl + C`。
- 看到询问时输入 `Y`，按回车。

## 5. 注册 GitHub 账号

1. 打开 https://github.com 。
2. 点击右上角 `Sign up`。
3. 输入邮箱。
4. 设置密码。
5. 设置用户名。
6. 按页面提示完成人机验证。
7. 打开邮箱，找到 GitHub 发来的验证邮件。
8. 点击邮件里的验证按钮。

## 6. 创建 GitHub 仓库

1. 登录 GitHub。
2. 点击右上角 `+`。
3. 点击 `New repository`。
4. `Repository name` 填写仓库名，例如：

```text
fxlocus-system
```

5. `Description` 可以填写：

```text
Open source FXLocus system deployment on Cloudflare Workers, D1 and R2.
```

6. 选择 `Public`，这样就是公开仓库。
7. 不要勾选 `Add a README file`。
8. 不要勾选 `.gitignore`。
9. 不要先添加 License，因为本项目已经带了 `LICENSE`。
10. 点击 `Create repository`。

## 7. 上传项目到 GitHub

### 方法 A：命令行上传

在 PowerShell 中运行：

```powershell
cd e:\fxlocus-system-open-source
git init
git add .
git commit -m "Initial open source system"
git branch -M main
git remote add origin https://github.com/你的用户名/你的仓库名.git
git push -u origin main
```

把 `你的用户名` 和 `你的仓库名` 改成 GitHub 上真实的内容。

如果 GitHub 要求登录，按提示登录即可。GitHub 现在通常不再使用账号密码直接推送，可能会让你用浏览器授权，或使用 Personal Access Token。

### 方法 B：GitHub Desktop 上传

如果你不想用命令行，可以用 GitHub Desktop。

1. 打开 https://desktop.github.com 。
2. 下载并安装 GitHub Desktop。
3. 打开 GitHub Desktop。
4. 点击 `Sign in to GitHub.com`。
5. 浏览器打开后授权登录。
6. 回到 GitHub Desktop。
7. 点击顶部菜单 `File`。
8. 点击 `Add local repository`。
9. 点击 `Choose`。
10. 选择：

```text
e:\fxlocus-system-open-source
```

11. 如果提示这不是 Git 仓库，点击 `create a repository`。
12. `Name` 填 `fxlocus-system`。
13. `Local path` 保持项目路径。
14. 点击 `Create repository`。
15. 左下角填写提交信息，例如：

```text
Initial open source system
```

16. 点击 `Commit to main`。
17. 点击上方 `Publish repository`。
18. 如果要公开开源，取消 `Keep this code private`。
19. 点击 `Publish Repository`。

上传后，打开 GitHub 仓库页面，应该能看到 `README.md`、`app`、`components`、`d1`、`docs` 等文件夹。

## 8. 注册 Cloudflare 账号

1. 打开 https://dash.cloudflare.com/sign-up 。
2. 输入邮箱和密码。
3. 按提示完成注册。
4. 登录 Cloudflare Dashboard。

如果你有自己的域名，可以在 Cloudflare 里添加站点；如果暂时没有域名，也可以先部署到 workers.dev 测试地址。

## 9. 登录 Wrangler

Wrangler 是 Cloudflare 的命令行工具。本项目已经把 Wrangler 放在依赖里，所以不用全局安装。

在项目目录运行：

```powershell
cd e:\fxlocus-system-open-source
npx wrangler login
```

执行后浏览器会打开 Cloudflare 授权页面。

1. 点击授权按钮。
2. 看到成功提示后回到 PowerShell。
3. 命令行显示登录成功即可。

验证：

```powershell
npx wrangler whoami
```

如果能看到你的 Cloudflare 账号信息，说明登录成功。

## 10. 创建 Cloudflare D1 数据库

D1 是系统数据库。

在项目目录运行：

```powershell
npx wrangler d1 create fxlocus-system
```

成功后会看到类似内容：

```toml
[[d1_databases]]
binding = "DB"
database_name = "fxlocus-system"
database_id = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
```

打开项目里的 `wrangler.toml`，找到：

```toml
database_id = "REPLACE_WITH_YOUR_D1_DATABASE_ID"
```

把它改成你刚刚得到的 `database_id`。

例如：

```toml
database_id = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
```

保存文件。

也可以在 Cloudflare 页面创建：

1. 打开 Cloudflare Dashboard。
2. 左侧进入 `Workers & Pages`。
3. 找到 `D1 SQL Database` 或 `D1`。
4. 点击 `Create database`。
5. 名称填 `fxlocus-system`。
6. 创建后进入数据库详情页，复制 Database ID。
7. 粘贴到 `wrangler.toml` 的 `database_id`。

## 11. 创建 Cloudflare R2 存储桶

R2 用来存储系统上传的图片、文档、交易截图、周报附件等。

命令行创建：

```powershell
npx wrangler r2 bucket create fxlocus-system-files
```

页面创建：

1. 打开 Cloudflare Dashboard。
2. 左侧点击 `R2 Object Storage` 或 `R2`。
3. 点击 `Create bucket`。
4. Bucket name 填：

```text
fxlocus-system-files
```

5. 点击创建。

确认 `wrangler.toml` 中有：

```toml
[[r2_buckets]]
binding = "R2_ASSETS"
bucket_name = "fxlocus-system-files"
```

## 12. 获取 R2 API 密钥

系统上传和下载文件时需要 R2 的 S3 API 密钥。

1. 打开 Cloudflare Dashboard。
2. 左侧进入 `R2 Object Storage`。
3. 找到 `Manage R2 API Tokens` 或 `API Tokens`。
4. 点击 `Create API token`。
5. 权限选择对象读写权限，例如 `Object Read & Write`。
6. Bucket 范围建议只选择 `fxlocus-system-files`。
7. 点击创建。
8. 创建后页面会显示：
   - Access Key ID
   - Secret Access Key
9. 立刻复制保存。Secret Access Key 只显示一次。

还需要 Account ID：

1. 在 Cloudflare Dashboard 右侧或账号首页找到 `Account ID`。
2. 复制它。
3. R2 Endpoint 按这个格式填写：

```text
https://你的_ACCOUNT_ID.r2.cloudflarestorage.com
```

## 13. 修改 wrangler.toml

打开 `wrangler.toml`，重点检查这些位置：

```toml
name = "fxlocus-system"

[vars]
APP_BASE_URL = "https://your-domain.com"  //你的域名
SYSTEM_COOKIE_DOMAIN = ""
D1_NAME = "fxlocus-system"
R2_BUCKET = "fxlocus-system-files"
R2_ENDPOINT = "https://YOUR_ACCOUNT_ID.r2.cloudflarestorage.com"  //你的R2id
R2_PUBLIC_BASE_URL = ""
R2_CDN_BASE_URL = ""
NEXT_PUBLIC_R2_CDN_BASE_URL = ""

[[d1_databases]]
binding = "DB"
database_name = "fxlocus-system"
database_id = "你的 D1 database_id"   //你的 D1 id

[[r2_buckets]]
binding = "R2_ASSETS"
bucket_name = "fxlocus-system-files"
```

修改规则：

- `APP_BASE_URL` 改成你的正式网站地址，例如 `https://system.example.com`。
- 暂时没有域名时，可以先填 Workers 地址，部署后再回来改。
- `SYSTEM_COOKIE_DOMAIN` 新手建议留空。只有同一个登录态要跨多个子域名时才填写，例如 `.example.com`。
- `R2_ENDPOINT` 改成你的 Account ID endpoint。
- `R2_PUBLIC_BASE_URL`、`R2_CDN_BASE_URL`、`NEXT_PUBLIC_R2_CDN_BASE_URL` 有 R2 公开访问域名时再填写。
- 如果你暂时没有配置 R2 公开域名，可以先保持空字符串，但音乐、公开图片预览、部分直链显示会受影响。

R2 公开域名可以这样设置：

1. 进入 Cloudflare Dashboard。
2. 打开 `R2 Object Storage`。
3. 点击 `fxlocus-system-files` 存储桶。
4. 找到 `Settings`。
5. 找到 `Public access` 或 `Custom domains`。
6. 可以启用 `r2.dev` 测试域名，或绑定自己的自定义域名。
7. 把得到的 `https://...` 地址填到上面的三个 R2 public/CDN 变量中。

## 14. 设置生产密钥

不要把真实密钥写进 GitHub。密钥应该放在 Cloudflare 的 Secrets 里。

命令行设置方式：

```powershell
npx wrangler secret put R2_ACCESS_KEY_ID
npx wrangler secret put R2_SECRET_ACCESS_KEY
npx wrangler secret put TRADE_LOG_RETENTION_SECRET
npx wrangler secret put SYSTEM_RETENTION_SECRET
npx wrangler secret put CONSULT_RETENTION_SECRET
```

每执行一条命令，终端会让你输入值。粘贴后按回车即可。

生成随机清理密钥：

```powershell
[guid]::NewGuid().ToString("N")
```

你可以执行三次，分别作为：

- `TRADE_LOG_RETENTION_SECRET`
- `SYSTEM_RETENTION_SECRET`
- `CONSULT_RETENTION_SECRET`

页面设置方式：

1. 打开 Cloudflare Dashboard。
2. 进入你的 Worker 项目。
3. 点击 `Settings`。
4. 点击 `Variables and Secrets`。
5. 点击 `Add`。
6. 类型选择 `Secret`。
7. 名称填写变量名，例如 `R2_SECRET_ACCESS_KEY`。
8. 值粘贴真实密钥。
9. 点击保存。

当前开源系统默认不需要 OpenAI API Key。如果你以后自己增加 AI 功能，再去对应 API 服务商后台创建 Key，并按新增功能要求放入 Cloudflare Secrets。

## 15. 初始化远程 D1 数据库结构

确认已经完成：

- `npx wrangler login`
- `wrangler.toml` 里已经填入正确的 `database_id`

然后运行：

```powershell
npm run d1:init:remote
```

这个命令会把 `d1/schema.sql` 写入 Cloudflare D1，并记录迁移版本。

如果看到 `[d1:init] Database schema is ready.`，说明数据库结构已经建好。

如果提示没有权限，重新运行：

```powershell
npx wrangler login
```

如果提示找不到数据库，检查：

- D1 数据库名称是不是 `fxlocus-system`。
- `wrangler.toml` 的 `database_id` 是否正确。
- 当前 PowerShell 是否在项目目录。

## 16. 创建第一个超级管理员

数据库结构创建好后，运行：

```powershell
$env:ADMIN_EMAIL="admin@example.com"  //设置你的超管账号
$env:ADMIN_PASSWORD="ChangeThisPassword123!"    //设置你的超管密码
$env:ADMIN_NAME="管理员"
npm run d1:create-admin:remote
```

把邮箱和密码改成你自己的。

密码至少 8 位，建议包含大小写字母、数字和符号。

成功后会看到：

```text
[d1:create-admin] Super admin is ready: admin@example.com
```

这个账号就是第一次登录系统的超级管理员。

## 17. 部署到 Cloudflare Workers

### 方法 A：命令行部署

适合已经能在电脑上运行命令的人。

```powershell
cd e:\fxlocus-system-open-source
npm run deploy
```

这个命令会先运行 Cloudflare 生产构建，再用 Wrangler 部署。

提醒：Windows 本地运行 OpenNext Cloudflare 构建可能会因为平台兼容问题失败。如果 Windows 本地失败，可以使用 Cloudflare GitHub 自动构建，或在 WSL、Linux、macOS 上运行。

### 方法 B：Cloudflare 连接 GitHub 自动部署

适合新手，也更适合开源项目。

1. 确认项目已经上传到 GitHub。
2. 打开 Cloudflare Dashboard。
3. 进入 `Workers & Pages`。
4. 点击 `Create` 或 `Create application`。
5. 选择 `Workers`。
6. 选择 `Import from Git` 或 `Connect to Git`。
7. 按提示连接 GitHub。
8. 选择你的仓库，例如 `fxlocus-system`。
9. Project name 填：

```text
fxlocus-system
```

10. Root directory 留空或填 `/`。
11. Build command 填：

```text
npm run cf:build
```

12. Deploy command 填：

```text
npx wrangler deploy
```

13. 如果页面要求 Node.js 版本，选择当前 LTS 版本。
14. 保存并开始部署。

Cloudflare 的构建环境是 Linux，更适合运行 `npm run cf:build`。

如果 Cloudflare 页面有环境变量设置，请把这些公开构建变量也填上：

```text
NEXT_PUBLIC_APP_VERSION=0.1.0
NEXT_PUBLIC_R2_CDN_BASE_URL=https://你的 R2 公开域名
NEXT_PUBLIC_LADDER_REFRESH_MS=60000
NEXT_PUBLIC_WEBRTC_TURN_URLS=
NEXT_PUBLIC_WEBRTC_TURN_USERNAME=
NEXT_PUBLIC_WEBRTC_TURN_CREDENTIAL=
```

Secrets 仍然按第 14 步放到 `Variables and Secrets` 的 Secret 中。

## 18. 绑定自己的域名

如果你有域名，例如 `example.com`，想让系统访问地址变成：

```text
https://system.example.com
```

操作：

1. 打开 Cloudflare Dashboard。
2. 进入你的 Worker。
3. 点击 `Settings`。
4. 找到 `Domains & Routes`。
5. 点击 `Add Custom Domain`。
6. 输入你的域名，例如 `system.example.com`。
7. 按页面提示确认 DNS。
8. 等待 Cloudflare 配置完成。

然后回到 `wrangler.toml`：

```toml
APP_BASE_URL = "https://system.example.com"
```

提交并重新部署。

如果使用 GitHub 自动部署：

```powershell
git add wrangler.toml
git commit -m "Configure production domain"
git push
```

Cloudflare 会自动重新构建。

## 19. 第一次登录

打开：

```text
https://你的域名/zh/system/login
```

输入第 16 步创建的邮箱和密码。

登录成功后会进入系统后台。

如果登录失败：

- 检查是否执行过 `npm run d1:init:remote`。
- 检查是否执行过 `npm run d1:create-admin:remote`。
- 检查账号邮箱是否拼写正确。
- 检查 Worker 是否绑定了正确 D1 数据库。
- 如果刚改过 `APP_BASE_URL` 或域名，重新部署一次。
- 新手建议 `SYSTEM_COOKIE_DOMAIN` 先留空。

```powershell
git status

git commit -m "Update system"
git push
```

如果你使用 Cloudflare GitHub 自动部署，推送后 Cloudflare 会自动构建新版本。

## 21. 本地 D1 说明

项目提供了本地 D1 命令：

```powershell
npm run d1:init:local
npm run d1:create-admin:local
```

但是 Windows 上 Wrangler 本地 D1 有时会因为本地运行时下载或兼容问题卡住。新手最稳定的路线是直接使用 Cloudflare 远程 D1：

```powershell
npm run d1:init:remote
npm run d1:create-admin:remote
```

等线上系统跑通后，再考虑本地 D1 调试。
